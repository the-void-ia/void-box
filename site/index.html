Compose. Isolate. Ship.

Composable agent runtime with hardware isolation

Skills are declared capabilities. They execute inside isolated micro-VM boundaries.

VoidBox = Agent(Skills) + Isolation

Get Started GitHub
rust 
use void_box::{VoidBox, Skill, Pipeline, Network};

// 5 lines to isolate an agent
let researcher = VoidBox::new("researcher")
    .skill(Skill::mcp("hackernews-api"))
    .prompt("Find today's top AI stories")
    .network(Network::outbound())
    .build()?;

let analyst = VoidBox::new("analyst")
    .skill(Skill::agent("claude-code"))
    .prompt("Extract key trends")
    .build()?;

let writer = VoidBox::new("writer")
    .skill(Skill::agent("claude-code"))
    .prompt("Draft a summary report")
    .build()?;

// compose into a pipeline — fresh VM per stage
let report = Pipeline::new()
    .pipe(researcher)
    .fan_out(vec![analyst, writer]) // parallel
    .run().await?;
hn_pipeline.yaml 
api_version: v1
kind: pipeline
name: hn-research

sandbox:
  memory_mb: 1024
  network: outbound

pipeline:
  boxes:
    - name: researcher
      prompt: "Find today's top AI stories"
      skills: ["mcp:hackernews-api"]

    - name: analyst
      prompt: "Extract key trends"
      skills: ["agent:claude-code"]

    - name: writer
      prompt: "Draft a summary report"
      skills: ["agent:claude-code"]

  stages:
    - type: box
      name: researcher
    - type: fan_out        # parallel
      boxes: [analyst, writer]
$ void-box run hn_pipeline.yaml

Why Choose VoidBox?

Agent runtime boundaries with the simplicity of declarative skills.

Hardware Isolated

Why stronger than shared-process agents?

Each run executes behind a micro-VM boundary, reducing cross-run blast radius.

Composable Runtime

Can I orchestrate multi-step agents?

Yes. Run single agents, pipelines, and workflows with explicit stage boundaries.

Skill-Native

How are capabilities declared?

MCP servers, SKILL files, and CLI tools are mounted as declared capabilities.

Policy Enforced

What controls exist at runtime?

Command allowlist, resource limits, seccomp, and controlled egress are applied in execution path.

Claude Code First-Class

What is the canonical runtime?

VoidBox always runs claude-code. Ollama support works through Claude-compatible provider mode.

Built-in Run Visibility

Can I inspect what happened?

Structured run events, logs, and session persistence make execution behavior inspectable.
